# Delaunay Maps GUI (Ap/Ac)

Приложение на Python с графическим интерфейсом для построения двух карт по точкам из Excel:

- карта значений `Ap`
- карта значений `Ac`

Карты строятся методом триангуляции Делоне по координатам `x`, `y`, используют общую шкалу (настраиваемый градиент) и позволяют проверить совпадение изолиний через прозрачное наложение. Отрисовка выполняется через **Matplotlib** во встроенном виджете PySide6.

## Возможности

- Загрузка файла Excel (`.xlsx`, `.xls`)
- Автоопределение колонок (`rn`, `x`, `y`, `Ap`, `Ac`) + ручной выбор
- Общая триангуляция Делоне для обеих карт
- Отдельные карты `Ap` и `Ac` с colorbar справа и числовыми делениями
- Режим `overlay` с регулировкой прозрачности смешивания Ap/Ac
- Опция принудительной зеркальности `Ap/Ac` для совпадения изолиний
- Плавные изолинии (интерполяция + сглаживание) с настройкой силы сглаживания
- Переключатели отображения точек и подписей координат
- Подписи названий точек из столбца `rn`
- Встроенная шкала масштаба на картах (в метрах; для градусов WGS84 — пересчёт в метры)
- **Спутниковая подложка** ([contextily](https://github.com/geopandas/contextily)): источник **Esri World Imagery** или **неофициальный Google Satellite** (URL тайлов; возможны ограничения ToS). Для координат **WGS84 в градусах**, без **поворота** и без «горизонтального выравнивания» (иначе снимок и тайлы Web Mercator перестают совпадать). Нужен **интернет**
- Регулировка **прозрачности заливки** поверх подложки (чтобы был виден снимок)
- Переключатель отображения изолиний (можно оставить только заливку)
- Инверсия осей `X`/`Y` и режим «поменять X и Y местами»
- Горизонтальное выравнивание карты (длинная сторона вдоль `X`)
- Ручной поворот карты в градусах (`-180° .. 180°`)
- Регулировка размера шрифта подписей точек (`rn` и координат)
- Регулировка отступа карты от рамки построения
- Live-перерисовка при смене чекбоксов и параметров
- Прокрутка левой панели и сворачивание блока параметров
- Экспорт текущего графика в PNG
- Кнопка `Экспорт Corel` (векторный экспорт в SVG/PDF/EPS; для Corel рекомендуется SVG)

Подложка в **Web Mercator (EPSG:3857)**; при её включении подписи осей дополняются «(м)».

## Структура проекта

- `app.py` - точка входа
- `ui/main_window.py` - GUI (PySide6)
- `core/data_loader.py` - чтение и подготовка данных Excel
- `core/interpolation.py` - триангуляция и уровни изолиний
- `core/basemap.py` - проекция WGS84 → Web Mercator и спутниковые тайлы
- `core/plotting.py` - отрисовка карт и overlay
- `requirements.txt` - зависимости

## Требования

## Использование

1. Нажмите `Загрузить Excel` и выберите файл с данными.
2. Проверьте соответствие колонок `rn/x/y/Ap/Ac` (или выберите вручную).
3. Нажмите `Построить карты Ap / Ac` для отдельных карт.
4. При необходимости включите спутниковую подложку (если доступна для ваших координат) и настройте прозрачность слоя над подложкой.
5. Нажмите `Проверить наложение (overlay)` для проверки совпадения изолиний.
6. Сохраните результат кнопкой `Сохранить график PNG`.
7. Для CorelDRAW используйте `Экспорт Corel` (оптимально в `SVG`).

### Подложка Esri / Google (неофиц.)

Экстент осей подгоняется к **квадрату** по большему размаху координат, чтобы при «узкой» вытянутой съёмке карта не превращалась в нечитаемую полоску; перед загрузкой тайлов для оси задаётся `aspect='auto'`, чтобы подложка не сжималась в «нитку».

Для **Google** тайлов запросы идут с заголовками, похожими на браузер (иначе сервер часто отклоняет стандартный `User-Agent` contextily). В браузере (как в других проектах с картой) тайлы загружаются иначе; здесь это всё равно неофициальный доступ по URL.

Google: неофициальный шаблон URL тайлов; возможны сбои и нарушение условий использования Google — используйте на свой риск. Условия Esri — на сайте Esri.

Макет фигуры без `constrained_layout` (ручные `subplots_adjust`), чтобы два colorbar и подложка не схлопывали оси в нулевую ширину.

Подложка недоступна при координатах не в градусах или при включённом «горизонтальном выравнивании».

## Live-перерисовка

График обновляется автоматически при смене большинства параметров; для ползунка прозрачности overlay и прозрачности слоя над подложкой перерисовка при отпускании мыши.

## Примечание по экспорту CorelDRAW

Прямой экспорт в `.cdr` из `matplotlib` недоступен.  
При выборе формата `CorelDRAW (*.cdr)` приложение сохраняет векторный `SVG` с тем же именем, который открывается в CorelDRAW.

## Формат входных данных

Ожидаются столбцы (имена могут отличаться, если выбрать вручную):

- `rn` - номер точки (опционально)
- `x` - координата X (долгота или восточное направление в проекции)
- `y` - координата Y (широта или северное направление)
- `Ap` - значение поля Ap
- `Ac` - значение поля Ac

Если имена столбцов уже совпадают с ожидаемыми (`rn`, `x`, `y`, `Ap`, `Ac`), поля в интерфейсе подставляются автоматически по умолчанию.

Минимум 3 уникальные точки по `(x, y)` обязательны для триангуляции.
