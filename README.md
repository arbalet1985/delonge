# Delaunay Maps GUI (Ap/Ac)

GUI-приложение на Python (PySide6 + Matplotlib) для построения двух карт по точкам из Excel:

- **карта значений `Ap`**
- **карта значений `Ac`**

Основа — **триангуляция Делоне** по координатам `x`, `y`. Карты используют общую шкалу и позволяют проверять совпадение изолиний через прозрачное наложение (**Overlay**).

## Возможности (кратко)

- **Импорт Excel**: `.xlsx`, `.xls`; автоопределение колонок + ручной выбор
- **Отдельные карты** `Ap` / `Ac` (2 панели) и **Overlay** для сравнения
- **Общий градиент** (2 цвета) или **плиточный градиент** (набор градаций)
- **Плавные изолинии** (интерполяция на сетку + сглаживание)
- **Инверсия осей X/Y**, режим **поменять X и Y местами**
- **Поворот карты** (в градусах)
- **Охват кадра по X/Y** (масштабирование `xlim/ylim`)
- **Спутниковая подложка** (тайлы) для координат WGS84 в градусах
- **Шкала масштаба** (в метрах) + опциональный **значок “Север (N)”**
- **Экспорт**: PNG и вектор (SVG/PDF/EPS; “Corel” сохраняет SVG)
- **Прокрутка** области карт: если холст больше окна — появляются полосы прокрутки

## Требования

- Python 3.10+
- Зависимости из `requirements.txt`

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python app.py
```

## Быстрый сценарий использования

1. Нажмите **`Загрузить Excel`** и выберите файл.
2. Проверьте сопоставление колонок **`x/y/Ap/Ac`** (и `rn`, если есть).
3. Нажмите **`Построить карты Ap / Ac`**.
4. Для проверки совпадения карт нажмите **`Проверить наложение (overlay)`**.
5. При необходимости включите **подложку**, настройте прозрачности.
6. Сохраните результат **`Сохранить график PNG`** или **`Экспорт Corel`**.

## Параметры и элементы интерфейса (подробно)

### Вкладки

- **Отдельные карты**: две панели `Ap` и `Ac`, обычно с colorbar справа.
- **Overlay**: совмещение `Ap`/`Ac` с настраиваемой прозрачностью.

> Холст вкладок помещён в область прокрутки: если вы увеличили ширину/высоту холста и он не влезает — появятся полосы прокрутки.

### Изолинии / заливка

- **Число уровней** или режим **«по шагу»** (если включён).
- **Плавные изолинии**: строит регулярную сетку и сглаживает её.
- **Показывать изолинии** / **подписывать изолинии** — управляют линиями и подписями.

### Цвет

Есть два режима:

1) **Градиент 2‑цветный**: кнопки `Цвет 1` / `Цвет 2`.

2) **Свои цвета по градациям** (плитки):

- **Градаций**: сколько полос/ступеней.
- **Плитки**: каждая плитка — отдельный цвет градации.
- **Подсветка изменённых плиток**: выделяются только те, где цвет меняли вручную.
- **ПКМ по плитке**:
  - **Сбросить эту плитку**
  - **Сбросить изменённые** (только вручную изменённые)
  - **Сбросить все плитки**
- **Градиент 1→N**: интерполирует цвета между “якорями” (1‑я, N‑я и вручную изменённые).

### Охват и размер (кадр и размер холста)

- **Поворот**: поворот содержимого карты (точки/изолинии/подложка) вокруг центра охвата.
- **Отступ рамки**: дополнительный запас вокруг данных.
- **Охват по X/Y** (в UI сейчас в блоке `Mercator: X/Y`):
  - больше % → шире кадр по соответствующей оси
  - меньше % → “приближение” по оси
  - работает как в Web Mercator, так и в обычных координатах
- **«Отдельные карты» / «Overlay»: Шир./Выс.**: размер холста Matplotlib в пикселях.

### Подложка (спутник/схема)

- Подложка включается только если координаты распознаны как **WGS84 в градусах**.
- При включении координаты переводятся в **Web Mercator (EPSG:3857)**.
- **Источник**: Esri/Yandex/Google (в т.ч. hybrid).
- **Слой / подложка**: прозрачность заливки поверх подложки (чтобы был виден снимок).
- **Сдвиг (E/N)**: сдвигает только подложку (данные остаются на месте) — подстройка тайлов.

#### Google API key (опционально)

Ключ для Google (если требуется выбранным источником) задаётся:

- в `config.local.yaml` (файл не коммитится) — пример в `config.example.yaml`
- или через переменную окружения `GOOGLE_MAPS_API_KEY` (имеет приоритет)

### Что показывать / Оси и данные

- **Точки**, **координаты**, **rn**, **координатная сетка**
- **Шкала масштаба по X/Y**
- **Север (N)**: значок ориентации на север в правом верхнем углу (можно выключить)
- **Инвертировать ось X/Y**, **поменять X и Y местами**
- **принудительная зеркальность Ap/Ac**

## Особенности и ограничения

### Подложка и координаты

- Подложка работает только для данных, похожих на **WGS84 в градусах** (`lon/lat`).
- При наличии подложки отрисовка ведётся в **EPSG:3857**, а подписи осей показываются в градусах.

### Тонкие “швы” на заливке поверх подложки

При прозрачной заливке поверх спутника Matplotlib может показывать тонкие линии на границах заливки. В проекте применены приёмы для уменьшения артефактов; если они заметны на вашем наборе данных/настройках — пробуйте:

- увеличить/уменьшить **прозрачность слоя**
- включить/выключить **плавные изолинии**
- изменить **размер сетки** / сглаживание

## Экспорт CorelDRAW

Прямой экспорт в `.cdr` из Matplotlib недоступен. При выборе `CorelDRAW (*.cdr)` приложение сохраняет **SVG** с тем же именем, который открывается в CorelDRAW.

## Формат входных данных

Ожидаются столбцы (имена могут отличаться, если выбрать вручную):

- `rn` — номер/имя точки (опционально)
- `x` — координата X (долгота или восточное направление в проекции)
- `y` — координата Y (широта или северное направление)
- `Ap` — значение поля Ap
- `Ac` — значение поля Ac

Если имена уже совпадают с ожидаемыми (`rn`, `x`, `y`, `Ap`, `Ac`), поля в интерфейсе подставляются автоматически.

Минимум **3 уникальные точки** по `(x, y)` обязательны для триангуляции.
